# Chip Bar v0.1 — Design

**Date:** 2026-05-16
**Status:** Approved (design phase)
**Scope:** Initial release of `chipbar` — macOS menu bar utility displaying live CPU, GPU and RAM utilisation for Apple Silicon machines.

---

## 1. Goals

- Show live CPU / GPU / RAM utilisation in the macOS system menu bar as a compact, two-row block.
- Update every 1 second by default; allow the user to switch to 2 seconds via a dropdown.
- Persist the chosen refresh interval across launches.
- Distribute as a Homebrew cask: `brew tap ihororlovskyi/tap && brew install --cask chipbar`.
- Keep the v0.1 surface area minimal — no extra metrics, no preferences window, no login items.

## 2. Non-goals (v0.1)

- Launch at login.
- Additional metrics (disk, network, temperature, power, ANE, etc).
- Refresh intervals other than 1 s and 2 s.
- Graphs, history, alerts, notifications.
- Localisation (UI is English-only).
- Notarisation or Developer ID signing — v0.1 ships ad-hoc signed.
- Universal binary — arm64 only.
- Intel Mac support.

## 3. Platform & toolchain

- **Target OS:** macOS 14.0 Sonoma and newer.
- **Architecture:** arm64 only (`ARCHS = arm64`, `EXCLUDED_ARCHS[sdk=*][arch=x86_64] = x86_64`).
- **Language:** Swift 5.10+.
- **UI framework:** AppKit (`NSStatusItem` + custom `NSView`). SwiftUI `MenuBarExtra` is rejected because precise two-row text rendering inside the 22 pt menu bar height needs explicit drawing control.
- **Build system:** Xcode project (`Chipbar.xcodeproj`).
- **Bundle id:** `com.ihororlovskyi.chipbar`.
- **App type:** Agent app — `LSUIElement = YES` in `Info.plist`. No Dock icon, no main window.

## 4. Architecture

Single-process app with four primary components:

```
                 ┌──────────────────┐
                 │   AppDelegate    │
                 │  (bootstrap)     │
                 └─────────┬────────┘
                           │
        ┌──────────────────┼────────────────────┐
        ▼                  ▼                    ▼
┌───────────────┐  ┌────────────────┐   ┌──────────────────┐
│  Preferences  │  │ MetricsSampler │   │ StatusBarCtrl    │
│ (UserDefaults)│  │ (actor + Task) │   │ (NSStatusItem +  │
│               │  │                │   │  NSMenu + view)  │
└───────┬───────┘  └────────┬───────┘   └────────▲─────────┘
        │                   │                    │
        │ interval change   │ AsyncStream        │
        └───────────────────┴────────────────────┘
                            │
              ┌─────────────┼───────────────┐
              ▼             ▼               ▼
        ┌──────────┐ ┌──────────┐  ┌────────────┐
        │CPUReader │ │RAMReader │  │ GPUReader  │
        │(Mach API)│ │(Mach API)│  │(IOReport)  │
        └──────────┘ └──────────┘  └────────────┘
```

### Components

- **AppDelegate.** Bootstraps the app on `applicationDidFinishLaunching`: constructs `Preferences`, `MetricsSampler`, `StatusBarController`, wires them, starts the sampling task.
- **Preferences.** Thin wrapper over `UserDefaults.standard`. One key: `chipbar.refreshIntervalSeconds: Int` (allowed values: `1`, `2`; default `1`). Exposes a Combine `AnyPublisher<Int, Never>` over `UserDefaults.publisher(for:)` so `AppDelegate` can observe changes and forward them to the sampler.
- **MetricsSampler.** A Swift `actor` running a long-lived `Task` that calls `CPUReader.sample()`, `GPUReader.sample()`, `RAMReader.sample()` once per interval, assembles a `Snapshot`, and yields it to an `AsyncStream<Snapshot>`. Exposes `setInterval(_ seconds: Int)` which cancels the current sleep and reschedules at the new cadence. Cancellation aware: terminates cleanly on `Task.cancel()`.
- **StatusBarController.** Owns the `NSStatusItem`, the custom `StatusBarView` (custom drawing), and the `NSMenu`. Consumes the sampler's `AsyncStream` on the main actor and forces a redraw on each new snapshot.
- **Readers.** Three independent metric sources, each behind a small protocol (`CPUReading`, `GPUReading`, `RAMReading`) for testability:
  - `CPUReader` — `host_processor_info(PROCESSOR_CPU_LOAD_INFO)`. Computes per-tick deltas of `user + system + nice` against `idle` between successive samples. First sample after launch returns `0%` (no prior state for delta).
  - `RAMReader` — `host_statistics64(HOST_VM_INFO64)`. Used ratio computed as `(internal + wired + compressed) / total`, matching Activity Monitor's "memory used" view rather than "free memory".
  - `GPUReader` — `IOReport` framework, channel `gpu_busy_state` (subsystem `pmp`). Same approach used by Stats.app, asitop and mactop. Private framework, runtime-permitted. First sample returns `0%`.

### Concurrency model

- `MetricsSampler` is an `actor` — serialises access to its internal state.
- The sampling loop runs in a single detached `Task` owned by the sampler.
- `StatusBarController` is `@MainActor`. The subscription to the sampler's `AsyncStream` runs on the main actor and updates AppKit directly.
- `Preferences` exposes changes via Combine (`UserDefaults.publisher(for:)`) or `AsyncStream`; sampler observes and reconfigures interval on change.

## 5. UI

### 5.1 Status bar view

Custom `NSView` subclass (`StatusBarView`) with fixed width (~64 pt) and the menu bar's intrinsic height (~22 pt). Drawn in two rows via `NSAttributedString.draw(in:)`:

```
[cpu][gpu][mem]    ← SF Symbols, ~8 pt, even spacing
 6%   9%   64%     ← values, monospaced digits, ~9 pt
```

- **Icons (top row):** SF Symbols `cpu` (CPU), `gpu.fill` (GPU), `memorychip` (RAM). All available on macOS 14+. Rendered with `NSImage(systemSymbolName:accessibilityDescription:)`.
- **Values (bottom row):** integer percentages, rendered with system font `.monospacedDigit` so column widths do not jitter as values change (e.g. `9%` → `10%`).
- **Colour:** `NSColor.labelColor` for both rows — adapts automatically to dark/light appearance.
- **Width:** fixed, so the menu bar layout to the left does not shift each second.
- **Update:** view caches the latest `Snapshot`; `setNeedsDisplay(_:)` is called on the main actor whenever a new snapshot arrives.

### 5.2 Dropdown menu

Opened on left click of the status item. Structure:

```
┌────────────────────────┐
│  CPU                6% │   (disabled info item)
│  GPU                9% │   (disabled info item)
│  RAM               64% │   (disabled info item)
├────────────────────────┤
│  Update every        ▸ │
│    ✓ 1 second           │
│      2 seconds          │
├────────────────────────┤
│  Quit Chip Bar         │
└────────────────────────┘
```

- Top three rows: read-only `NSMenuItem`s displaying full values. Useful when the bar text is too small to read precisely. Refreshed whenever the menu is about to be shown (`menuWillOpen`) and on each snapshot if the menu is open.
- `Update every` is an `NSMenuItem` with a submenu containing two items (`1 second`, `2 seconds`). The selected one has `state = .on` (checkmark). Selecting writes to `Preferences`, which propagates to the sampler.
- `Quit Chip Bar` calls `NSApp.terminate(nil)`.

## 6. Persistence

- `UserDefaults.standard`, single key `chipbar.refreshIntervalSeconds` (Int).
- Default `1` when missing.
- Validation: any value other than `1` or `2` is treated as `1`.
- Read on launch. Written on every menu selection. `Preferences` publishes interval changes via Combine; `AppDelegate` subscribes and forwards each change to `MetricsSampler.setInterval(_:)`.

## 8. Testing

All tests run via `xcodebuild test`. No test touches real Mach or IOReport APIs — every reader has an injectable protocol and tests use mock implementations.

- `CPUReaderTests`
  - Two synthetic `host_cpu_load_info_data_t` snapshots → expected delta percentage.
  - All ticks in `idle` → 0%.
  - All ticks in `user` → 100%.
  - Zero delta between two equal snapshots → 0% (no division by zero).
  - First call returns 0% (no prior state).
- `RAMReaderTests`
  - Synthetic `vm_statistics64_data_t` → expected `(internal + wired + compressed) / total`.
  - Edge case: pages outside the formula (free, speculative) are excluded.
- `GPUReaderTests`
  - `protocol GPUReading` mocked to return scripted values.
  - First sample returns 0%.
  - Zero delta handled.
- `PreferencesTests`
  - Use `UserDefaults(suiteName: "test-...")` instances per test for isolation.
  - Default is 1 when key absent.
  - Out-of-range values (0, 3, -5) fall back to 1.
  - Write → read round trip.
- `MetricsSamplerTests`
  - Inject mocked readers.
  - Subscribing to `AsyncStream` produces snapshots at expected cadence (use a virtual `Clock` if available, otherwise tolerance-based timing).
  - Changing interval mid-flight reschedules and emits at new cadence.
  - Cancellation terminates the loop.

## 9. Distribution

### 9.1 Release build

Script `scripts/build-release.sh`:

1. `xcodebuild archive -scheme Chipbar -archivePath build/Chipbar.xcarchive -configuration Release`
2. `xcodebuild -exportArchive` with an `ExportOptions.plist` (method: `mac-application`, signing: manual / none) → `build/Chipbar.app`
3. `codesign --force --deep --sign - build/Chipbar.app` — ad-hoc signing.
4. `ditto -c -k --keepParent build/Chipbar.app build/Chipbar-vX.Y.Z.zip`
5. `shasum -a 256 build/Chipbar-vX.Y.Z.zip` → captured for the cask.

### 9.2 Homebrew cask

`ihororlovskyi/homebrew-tap/Casks/chipbar.rb`:

```ruby
cask "chipbar" do
  version "X.Y.Z"
  sha256 "..."

  url "https://github.com/ihororlovskyi/chipbar/releases/download/v#{version}/Chipbar-v#{version}.zip"
  name "Chip Bar"
  desc "Tiny macOS menu bar monitor for Apple Silicon (CPU/GPU/RAM)"
  homepage "https://github.com/ihororlovskyi/chipbar"

  depends_on macos: ">= :sonoma"
  depends_on arch: :arm64

  app "Chipbar.app"

  zap trash: [
    "~/Library/Preferences/com.ihororlovskyi.chipbar.plist",
  ]
end
```

User commands:

```
brew tap ihororlovskyi/tap
brew install --cask chipbar
```

### 9.3 Gatekeeper

Because the binary is only ad-hoc signed (no Developer ID, no notarisation), first launch will be blocked by Gatekeeper. README documents two-step workaround: right-click `Chipbar.app` → Open, then confirm; or System Settings → Privacy & Security → "Open Anyway". Notarisation is deferred to a later release.

## 10. CI

Two GitHub Actions workflows under `.github/workflows/`.

### 10.1 `ci.yml` — test-only

- Trigger: `pull_request` and `push` to `main`.
- Runner: `macos-14` (Apple Silicon).
- Steps:
  - `actions/checkout@v4`
  - `xcodebuild test -scheme Chipbar -destination 'platform=macOS,arch=arm64' -enableCodeCoverage YES`

### 10.2 `release.yml` — full release on tag

- Trigger: `push` of tag matching `v*`.
- Runner: `macos-14`.
- Steps:
  - `actions/checkout@v4` with `fetch-tags: true`.
  - Run tests (same command as `ci.yml`).
  - `scripts/build-release.sh` (uses the tag as version).
  - `gh release create ${{ github.ref_name }} build/Chipbar-${{ github.ref_name }}.zip --generate-notes` (uses `GITHUB_TOKEN`).
  - Compute `sha256` of the zip.
  - Checkout `ihororlovskyi/homebrew-tap` with token `GH_TAP_PAT`.
  - `scripts/update-cask.sh <version> <sha256>` — `sed` `version` and `sha256` in `Casks/chipbar.rb`.
  - `git commit` and `git push` to the tap.

### 10.3 Secrets

- `GITHUB_TOKEN` — provided automatically, used to create the release.
- `GH_TAP_PAT` — fine-grained Personal Access Token, scope: `ihororlovskyi/homebrew-tap` only, permissions: Contents read/write. Created manually in GitHub UI and added to `chipbar` repo secrets.

## 11. Open follow-ups (post v0.1)

- Notarisation + Developer ID signing → remove Gatekeeper friction.
- Launch at login via `SMAppService`.
- Additional metrics: disk, network, temperature, power.
- More refresh intervals or a free-form input.
- Hover tooltip on the status item showing exact values.
- Universal binary for Intel users (separate code path for GPU on Intel).
